# Lesson 14 - Capstone Implementation and Review

## Goal

Execute the full SDD + TDD + BDD workflow on your chosen block.

## Required outputs

- final specification
- final test plan
- RTL source
- testbench source
- BDD scenarios
- assertion ideas or implementations
- regression results summary
- design reflection note

## Reflection prompts

- Which requirement was most ambiguous at first?
- Which bug would likely have been missed without test planning?
- Which scenario was easiest to misunderstand before BDD wording?
- What would you improve in a second version?

## Final success criteria

A successful capstone is not the biggest RTL block. It is the block with the clearest engineering traceability from requirement to verification.
