# Lesson 13 - Capstone Architecture and Planning

## Goal

Choose one digital block and define it fully before implementation.

## Recommended project choices

1. synchronous FIFO
2. round-robin arbiter
3. programmable counter controller
4. request / acknowledge controller

## Deliverable package

Your capstone package should include:
- module overview
- port table
- reset behavior
- timing rules
- legal / illegal scenarios
- corner cases
- test plan matrix
- BDD scenario sheet
- implementation plan

## Review questions

- Is every rule observable in simulation?
- Are all boundary conditions named?
- Could another engineer implement this block from your document?

## Advice

Do not start RTL too early. The capstone is meant to reward good planning.
