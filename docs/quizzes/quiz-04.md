# Quiz 04

## Multiple choice

1. Which statement best describes a strong capstone deliverable?
   - A. the longest RTL code
   - B. the most clever always block
   - C. clear traceability from spec to tests to implementation
   - D. the highest number of comments only

2. Which is a good assertion candidate?
   - A. signal names should be short
   - B. grant should never be two-hot when only one request may be served
   - C. the RTL should look elegant
   - D. simulation should be fast

## Short answer

3. Why is requirement traceability important in digital IC design?
4. Give one example where a test is useful and one example where an assertion is useful.

## Answer key

1. C
2. B
3. It ensures every intended behavior is protected and reviewable.
4. Example answers: test for specific scenario sequencing, assertion for invariant or forbidden condition.
