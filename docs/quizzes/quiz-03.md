# Quiz 03

## Multiple choice

1. Which statement best matches BDD in hardware?
   - A. it replaces simulation
   - B. it expresses behavior as scenarios like Given / When / Then
   - C. it removes the need for specifications
   - D. it only works for software APIs

2. Which is the strongest BDD scenario?
   - A. Given normal operation, when something happens, then output is okay
   - B. Given FIFO is empty, when rd_en is asserted, then empty remains high and underflow handling occurs
   - C. Given design is valid, when user tests it, then it should work
   - D. Given reset, when system starts, then everything is normal

## Short answer

3. How can BDD help RTL and verification teams communicate better?
4. Why are assertions helpful after refactoring?

## Answer key

1. B
2. B
3. It makes expected behavior explicit, reviewable, and easy to translate into tests.
4. They preserve design intent and catch accidental behavior changes.
