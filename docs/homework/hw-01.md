# Homework 01 - Rewrite a Weak Requirement into a Hardware Spec

## Assignment

Given this weak requirement:

> Design a counter that can count and show when it reaches the limit.

Rewrite it into a proper hardware specification.

## Your output must include

- module purpose
- interface definition
- reset behavior
- increment behavior
- limit behavior
- output timing behavior
- corner cases

## Submission checklist

- at least 8 bullet-point requirements
- at least 3 corner cases
- no ambiguous words like "normal", "proper", or "fast"

## Stretch goal

Add one requirement that would be easy to forget during implementation.
