# SDD / TDD / BDD for Verilog and Digital IC Design

Welcome. This course is designed for RTL designers, verification engineers, FPGA developers, and digital IC learners who want to move beyond code-first design and adopt a more systematic engineering flow.

## What you will learn

By the end of this course, you will be able to:

- write clear hardware specifications before coding RTL
- derive test plans directly from requirements
- apply TDD thinking to small and medium RTL modules
- use BDD style scenarios to clarify behavior and corner cases
- refactor RTL while protecting behavior with tests and assertions
- build a spec-driven workflow for FIFO, arbiter, counter, and controller style modules

## Who this course is for

- Verilog beginners who already know syntax but want a stronger design process
- RTL engineers who want more predictable quality
- verification engineers who want tighter linkage between spec and testbench
- FPGA learners building reusable design habits

## Course style

This course is taught like a high-quality online course:

- structured lessons
- learning goals per chapter
- quiz after each learning block
- homework with design tasks
- capstone project

## Recommended background

You should already know:

- basic combinational vs sequential logic
- always blocks, assign, reg / wire or logic
- how to compile and simulate simple Verilog

## Final outcome

You will produce a mini portfolio of:

- a spec for a hardware block
- a test plan
- a TDD-style RTL implementation
- a BDD-style behavior checklist
- a regression-ready verification flow
