# SDD / TDD / BDD for Verilog and Digital IC Design

> A premium-style practical course for RTL engineers who want to design with more clarity, test with more confidence, and review hardware behavior like a system engineer instead of a syntax typist.

## Course promise

Most Verilog learners are taught how to write RTL.
Very few are taught how to **think before RTL**, how to **derive tests from requirements**, and how to **protect behavior while refactoring**.

This course fixes that.

By the end of this program, you will be able to:

- turn ambiguous design requests into concrete hardware specifications
- derive test plans directly from requirements
- apply TDD thinking to RTL implementation
- use BDD scenarios to communicate behavior clearly across design and verification teams
- use assertions and regression as engineering safety rails
- build a reusable development flow for counters, FIFOs, arbiters, and control logic

---

## Who this course is for

### Ideal students
- RTL engineers who already know Verilog syntax but want a stronger process
- FPGA developers who want to reduce rework and late bug discovery
- digital IC engineers preparing for deeper design reviews
- verification engineers who want better requirement traceability
- engineering leads who want a cleaner team workflow

### Not ideal for
- absolute beginners who have never written Verilog
- people looking only for syntax tutorials
- analog / mixed-signal specialists with no RTL background

---

## What makes this course different

### 1. It is not syntax-first
This is not another course that teaches `always @(posedge clk)` and stops there.

### 2. It is design-process-first
You will learn how professional engineering discipline reduces ambiguity before bugs are created.

### 3. It bridges design and verification
The same requirement will be used to drive:
- specification
- test planning
- RTL implementation
- assertions
- review conversations

### 4. It is hardware-native
The material is adapted specifically for digital design, not copy-pasted from software development theory.

## Course structure

### Part 1. Foundation
Build the mental model for why SDD, TDD, and BDD matter in digital IC work.

### Part 2. Spec-Driven Design
Learn how to write hardware specifications that are testable and reviewable.

### Part 3. Test-Driven RTL
Learn how to implement modules with smaller, safer verification loops.

### Part 4. Behavior-Driven Validation
Translate hardware behavior into human-readable scenarios and executable expectations.

### Part 5. Capstone
Apply the full workflow to a realistic digital design block.

## Recommended study method

To get the most value:

1. read each lesson in order
2. answer the quiz without looking at the key first
3. do the homework in writing, not only mentally
4. implement the small RTL exercises locally
5. keep a design notebook for your own specs and test plans
